## 🌟 1. 这个模块是干什么的？（大白话版）

想象你在经营一家游戏机租赁店。当客户走进来大喊一声时，AI 就像是一个**“前台分流员”**：
- 如果客户问：**“多少钱一天？”** → AI 识别意图为 `rule`（规则），带他去看价目表。
- 如果客户问：**“有没有黑神话悟空？”** → AI 识别意图为 `game`（游戏），去仓库查库存。
- 如果客户说：**“你好啊！”** → AI 识别为 `all`（通用），礼貌回个话。

**核心意义：** 它是整个智能客服的“第一站”，只有听懂了意图，后面的 RAG（检索增强生成）和大模型才知道该去哪查资料，避免“答非所问”。

## 📂 2. 文件夹里装的都是啥？

```text
RAG_Intend_Bert/
├── config/                 # 🛠️ 【配置中心】：像遥控器一样，改参数（如学习率、路径）只需动这里，不用改核心代码。
├── data/                   # 📝 【资料库】：
│   ├── row/                # 原始的“聊天记录” (intends.json)。
│   └── processed/          # AI 真正学习用的“教科书” (train.csv)。
├── src/                    # 🏗️ 【工厂】：把复杂的“造车”过程封装起来，你只需要下命令 get_model() 就能得到一门模型。
├── model/                  # 🧠 【大脑】：训练好的模型就存在这，它是 AI 学习后的“记忆”。
├── train.py                # 🏃‍♂️ 【教练】：运行它，AI 就开始对着“教科书”拼命练习。
└── test_model.py           # ⚖️ 【考官】：运行它，考考 AI 到底变聪明了没。
```

## 🚀 3. 三大面试“杀手锏”核心技术

### ⚡ 亮点 A：为什么选 DistilBERT 而不是 BERT？
*   **面试标准回答：** “我们在实际场景中对比了 BERT 和 DistilBERT。BERT 虽然强大，但在 CPU 上推理会有明显的延迟。**DistilBERT 保持了 97% 的精度，但速度快了 60%**。对于租赁客服这种需要秒回的场景，极致的响应速度比那 3% 的精度提升更重要。”

### 🏗️ 亮点 B：代码架构中的“工厂模式”
*   **面试标准回答：** “为了让项目更具工程性，我没有把模型加载代码写死在脚本里，而是封装了一个 `ModelFactory`。这样无论我是要进行**模型训练**还是**在线推理**，都只需要调用同一个工厂接口。这种解耦设计让代码非常容易维护和扩展。”

### 🧪 亮点 C：如何评价 AI 好不好使？（F1-Score）
*   **面试标准回答：** “在评估模型时，我没有只看‘准确率（Accuracy）’。因为如果我的数据里 90% 都是问规则的，AI 哪怕全猜规则也能拿 90 分。所以我使用了 **F1-Score (加权平均)**，它综合考虑了‘查得准不准’和‘查得全不全’，能真实反映 AI 在各类意图上的表现。”

## 🛠️ 4. 怎么把这个项目跑起来？

### 第一步：装环境
```bash
pip install -r requirements.txt
```

### 第二步：教 AI 学习
运行 `train.py`。你会看到屏幕上在跳动数字，那是 AI 在学习。
```bash
python train.py
```

### 第三步：去考考它
运行 `test_model.py`，然后输入一句话（比如：我想租 Switch），看它能不能猜对。
```bash
python test_model.py
```

## 🏁 5. 复试常见 Q&A（提前背诵）

*   **Q：你的训练数据哪里来的？**
    *   *A：我们根据租赁业务场景（规则、游戏、服务）模拟了约 1500 条语料，并参考了开源的对话数据集进行数据增强。*
*   **Q：模型过拟合（死记硬背）了怎么办？**
    *   *A：我在配置里设置了 `Weight Decay`（权重衰减）和 `Warmup`（预热学习率），并且只保存评估指标最好的那一轮模型。*
*   **Q：如果用户说的话意图很模糊怎么办？**
    *   *A：程序会输出每个类别的‘置信度得分’（Confidence Scores）。如果最高分都低于阈值，我们会将其归类为通用意图，或引导用户补充信息。*
